### Как работает рендер

1. `src/data/services.ts` экспортирует массив `services: Service[]`
2. `src/components/Services.astro` на сервере группирует услуги по `category` и рендерит 4 панели (tab-panel)
3. Активная по умолчанию панель «Ресницы» рендерится без `display: none`, остальные скрываются через inline `style="display: none"`
4. Карточки показывают превью (thumbnail + название + цена)
5. По клику на карточку открывается единый fullscreen modal с фото, описанием, ценой и кнопкой «Записаться»
6. Modal закрывается: клик по фону, кнопка ✕, клавиша Escape
7. Переключение табов: JS listener на `#servicesTabs` вызывает `showTab(tab)` — показывает нужную панель через `element.style.display = ''`, скрывает остальные
8. При загрузке страницы JS init вызывает `showTab('lashes')` для синхронизации состояния

### Изображения услуг

- Базовая папка: `public/assets/img/services/`
- Для каждой услуги: отдельная подпапка (slug услуги) + один файл изображения внутри
- В `services.ts` поле `image` указывает путь вида: `/assets/img/services/{slug}/{filename.ext}`

## Правило работы с UI/UX

Для задач, связанных с интерфейсом, визуальной системой, паттернами взаимодействия, карточками, вкладками, модалками, формами и responsive-поведением, агент должен использовать skill `ui-ux-pro-max` до внесения изменений в код.

Порядок работы:
1. Сначала UX/UI-анализ текущего состояния.
2. Затем короткий план паттерна/решения.
3. Только после утверждения — реализация.

Не фантазировать произвольно по UI-поведению, если решение можно опереть на `ui-ux-pro-max`.
